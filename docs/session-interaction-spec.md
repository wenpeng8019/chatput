# 会话交互页面 — UE 交互规格说明

本文件描述会话交互页面的视觉结构、手势交互、阈值参数与触感反馈规格，作为 iOS / Android 双端对齐的依据。涵盖语音输入、文字输入、光标控制、远程屏幕幕布等全部交互层。

相关文档：[远程桌面缩放闪烁问题](remote-screen-resize-flash.md)

---

## 1. 页面结构

页面从上到下分为四个区域：

```
┌───────────────────────────────────────────┐
│ ① Header 区                                │  标题栏 + 更多菜单
│   "ChatPUT"                           ⋮   │
├───────────────────────────────────────────┤
│ ② 远程桌面窗口（下拉幕布，收起时隐藏）      │  从顶部下拉覆盖
│   视频画面 + 小地图 + 握把                  │  §9 详述
├───────────────────────────────────────────┤
│ ③ 输入会话列表                             │  桌面焦点窗口的历史会话
│   [Safari - 搜索]                          │  自动创建 / 关闭 / 切换
│   [VS Code - main.go]                      │
├───────────────────────────────────────────┤
│ ④ 交互操控区                               │  §2–§5, §11 详述
│   hint 提示文字                             │
│   ┌─────────────────────────────────┐      │
│   │  ⌫ (退格/Esc)  🎤 (语音/Dpad)  ⏎  │
│   └─────────────────────────────────┘      │
└───────────────────────────────────────────┘
```

### 区域说明

| 区域 | 名称 | 说明 |
|------|------|------|
| ① | Header 区 | 显示设备名 / 房间号，右侧「更多」按钮弹出操作菜单（查看屏幕、撤销、全选、清空）。详见 §12 |
| ② | 远程桌面窗口 | 下拉幕布式视频面板，收起时完全隐藏（`GONE`），不影响下方布局。详见 §9 |
| ③ | 输入会话列表 | 桌面端每个焦点输入框对应一个会话，自动创建/关闭，支持切换 |
| ④ | 交互操控区 | 核心交互区，承载**文本交互模式**和**方向交互模式**两种操控模式（见 §2） |

## 2. 交互操控区模式

交互操控区（区域 ④）根据桌面端输入控件的可用状态，在两种模式间自动切换：

```
交互操控区
├── 文本交互模式（inputAvailable = true）
│   ├── 语音输入模式  默认：按住说话 + 拖动控光标
│   └── 文本输入模式  上拉呼出键盘输入
│
└── 方向交互模式（inputAvailable = false）
    └── D-pad 导航  方向键移动光标 + Esc/Enter
```

### 2.1 文本交互模式

桌面端输入框可用的正常模式。交互区三个按钮分别为：退格、语音、回车。

| 子模式 | 触发方式 | 详见 |
|--------|----------|------|
| 语音输入（默认） | 页面进入时自动进入 | §3–§5 |
| 文本输入 | 上拉或点击把手区域 | §10 |

### 2.2 方向交互模式（D-pad 模式）

桌面端输入框暂时不可用（如 AI 弹出确认菜单遮住输入框）时，桌面通过 WebRTC 发送 `session-input-lost` 消息，手机自动切换到方向模式。

交互区三个按钮重新定义为：

| 按钮 | 文本模式功能 | 方向模式功能 |
|------|-------------|-------------|
| 中间 🎤 | 按住说话 + 拖动控光标 | D-pad 四向点击 → `cursorUp/Down/Left/Right` |
| 左侧 ⌫ | 退格（tap）/ 清空（长按） | Esc — `escape` |
| 右侧 ⏎ | Shift+Enter（tap）/ Enter（长按） | Enter — `enter` |

方向模式下的 Enter 和 Esc 命令通过 DataChannel 的 `action` 消息发送至桌面端，由 `TextInjector` 注入对应按键。详见 §11。

## 3. 语音输入模式 — 视觉结构

```
            ┌─────────────────────────────────────────┐  ← 顶部圆角弧线装饰（左右各一）
            │  ⌫            🎤            ⏎            │
            │ 退格/清空    语音/光标    回车/Enter     │
            └─────────────────────────────────────────┘
              面板圆角 34，浅色 surface，1px 描边
```

- **面板**：圆角 `34`，背景 `surface`，描边 `line`，水平内边距 `20`、垂直内边距 `13`。
- **顶部圆角弧线装饰（CornerArc）**：面板左上 / 右上各一条同心弧线，半径等于面板 `cornerRadius = 34`，曲率与面板边缘完全一致；描边色 `textTertiary @ 0.28`，线宽 `1.8`，尺寸 `34×34`，右侧镜像。仅装饰，不参与命中测试。**此为 iOS 方案，圆角曲率与角落装饰存在平台差异，详见 §8。**
- **提示文字（hint）**：位于面板上方，默认「按住说话」；随面板一起上移淡出。
- 面板无可见把手横线——与 Android 一致（Android 的 `drag_handle` 是不可见的全宽感应层）。

### 三个按钮

| 位置 | 按钮 | 轻点（tap） | 长按（hold 0.75s） |
|---|---|---|---|
| 左 | 退格 `delete.left` | `backspace` | `clear`（清空，toast「已清空」） |
| 中 | 语音 `mic.fill` | 见 §4 | 见 §4 |
| 右 | 回车 `return` | `shiftEnter` | `enter` |

### 按钮与图标尺寸（标准）

| 元素 | 容器 | iOS 图标 | Android 图标 | 说明 |
|---|---|---|---|---|
| 左右按钮（退格 / 回车） | `56` | `font(size: 22)` | `27dp` | iOS SF Symbol 字号，Android 矢量图 dp |
| 中间麦克风 | `78` | `font(size: 30)` | `34dp` | 同上 |

> 两端容器尺寸一致（56 / 78）。图标标称数值不同是有意为之：SF Symbol 在同一字号下渲染字形比同等 dp 的矢量图更饱满，Android 矢量图 `24×24` viewport 自带留白，因此需更大 dp 值使可见字形对齐。

### 长按进度按钮（HoldProgressButton）
- 按下即开始 `0.75s` 进度环动画（`holdDuration = 0.75`）。
- 满 `0.75s` 触发 `onHold`，伴随成功触感。
- 未满时松手触发 `onTap`，伴随轻触感。
- 手指滑出按钮范围视为取消。
- 按下时按钮缩放至 `~0.94`。

## 4. 中间语音按钮手势

中间按钮承载「按住说话 + 拖动控光标 + 上下切行 + 双击英文」多合一手势，基于单个 DragGesture。

### 4.1 按住说话
- 按下立即开始录音（`startTalking`），轻触感，hint 变「正在听…」。
- 松手：
  - 按压时长 `< 0.18s`（`tapMaxDuration`）→ 视为轻点，取消录音，进入双击判定。
  - 否则 → 停止录音并提交识别结果。

### 4.2 双击进入英文模式
- 轻点后 `0.35s`（`doubleTapMax`）内再次轻点，且英文识别可用 → 切换英文模式，hint「请说英文」。
- 单次轻点 → toast「再按一次进入英文输入」。

### 4.3 拖动进入光标模式（防误触保护区）
- **激活阈值**：位移 `hypot(dx, dy) > 28`（`cursorActivation = 28`）才切到光标模式。
  - 与 Android 对齐：Android 为 `max(scaledTouchSlop × 2, 28dp)`，典型即 `28dp`。
- **锁轴**：进入时按主方向锁定轴；垂直需明显占优（`|dy| > |dx| × 1.8`）才锁为上下行，否则默认左右移光标。
- 进入光标模式时丢弃录音，刚性触感，麦克风保持蓝色，hint 切换为方向提示。

### 4.4 水平：棘轮式移动光标

#### 阶段 A — 棘轮逐字（精确）
- 步长 `cursorStep = 24`：拖动距离每累计 24 发送一次 `cursorLeft/Right`。每步轻触感。
- 基线以进入光标模式的位置为起点。

#### 阶段 B — 连续自动移动（越过侧按钮中心后）
- 触发点：麦克风中心到侧按钮中心距离 = `面板内容宽 / 2 − 侧按钮半径(28)`，动态自适应。
- 过阈后按时间自动重复，间隔随偏移从 `0.2s`（5次/s）线性加速到 `0.04s`（25次/s）。

### 4.5 垂直：上下切行（swipe，非滑动）
- 一次 swipe 只切一行（离散触发，与水平棘轮不同）。
- 松手瞬间判定：锁定轴偏移 `≥ 32`（`swipeTrigger`）→ 发送一次 `cursorUp/Down`。
- 不足 `32` → 不切行。

### 4.6 按钮动效

#### 中间语音球
| 动效 | 标准 |
|------|------|
| 按下缩放 | 缩小到 ~0.93，约 130–150ms 缓出 |
| 光晕 | 录音时外圈光晕放大浮现，停止时缩回淡出 |
| 进入光标模式 | 球保持蓝色、光晕收起；方向由四周提示装饰表达 |
| 阴影 | 平台差异：iOS 柔和投影，Android 扁平 |

#### 左右长按按钮进度环
- 旋转进度环半径与大按钮一致（直径 78），环溢出按钮本体居中绘制。
- 环线宽 `ringSize / 20 ≈ 3.9`，从正上方顺时针填充。
- 时长 `holdDuration = 0.75s`。

### 4.7 方向提示装饰（directionHints）

麦克风四周的一组淡蓝装饰，仅视觉不参与命中测试。

#### 构成
- 四向扁角箭头（chevron）+ 左右圆点。

#### 状态联动
| 状态 | 上下箭头 | 左右箭头 + 圆点 |
|------|---------|----------------|
| 空闲 | 显示 | 显示（单点） |
| 水平移光标 | 隐藏 | 显示 |
| 连续触发 | 隐藏 | 第二点淡入 + chevron 外移 |
| 垂直切行 | 显示 | 隐藏 |
| 录音中 | 整体淡出 | 整体淡出 |

#### 视觉规格（两端对齐）
| 项 | iOS | Android |
|---|---|---|
| 色彩 | `accent @ opacity 0.34` | `chatput_accent`，`baseAlpha 0.34` |
| chevron 线条 | SF Symbol `chevron.compact.*` | `Path` 描边，`strokeWidth 1.5dp` |
| 圆点 | `Circle` 直径 `3` | `drawCircle` 半径 `1.5dp` |
| 状态过渡 | spring 动画 | `ValueAnimator 160ms` |

## 5. 关键参数（TalkUX）

| 参数 | 值 | 含义 |
|---|---|---|
| `cursorActivation` | `28` | 进入光标模式的最小拖动距离 |
| `cursorStep` | `24` | 水平棘轮步长（每字符） |
| `swipeTrigger` | `32` | 上下切行的松手触发距离 |
| `verticalBias` | `1.8` | 锁定上下轴的垂直占优倍数 |
| `continuousThreshold` | 动态（回退 `96`） | 进入连续移动的偏移阈值 |
| `repeatSlow` | `0.2s` | 连续移动最慢间隔 |
| `repeatFast` | `0.04s` | 连续移动最快间隔 |
| `tapMaxDuration` | `0.18s` | 轻点最大按压时长 |
| `doubleTapMax` | `0.35s` | 双击判定最大间隔 |
| `composerLift` | `56` | 上拉切换时面板上移位移 |
| `holdDuration` | `0.75s` | 长按按钮触发时长 |

## 8. 平台差异

以下两项在两端有意保持不同，属于各平台原生质感差异。

### 8.1 面板圆角曲率
| 平台 | 类型 |
|------|------|
| iOS | `RoundedRectangle(cornerRadius: 34, style: .continuous)` — 连续曲率 |
| Android | `MaterialCardView` `cardCornerRadius = 34dp` — 标准圆弧 |

### 8.2 角落装饰
| 平台 | 方案 |
|------|------|
| iOS | 左上/右上 `CornerArc` 同心弧线（`textTertiary @ 0.28`） |
| Android | 无弧线装饰，面板依靠 `1dp` 全周描边 |

### 8.3 更多按钮图标
| 平台 | 图标 | 朝向 |
|------|------|------|
| iOS | `ellipsis` (SF Symbols) | 横向 `⋯` |
| Android | `more_vert` (Material) | 纵向 `⋮` |

## 9. 远程桌面窗口 — 下拉幕布

从屏幕顶部下拉的悬浮视频面板，覆盖标题区和消息列表，与交互操控区共存。

> **iOS 现状**：尚未实现。

### 9.1 形态

```
   ┌──────────── 视频面板（screen_panel） ────────────┐  ← 顶部延展到状态栏下
   │  ▓▓▓▓▓▓▓▓  远程桌面画面（SurfaceViewRenderer） ▓▓│
   │  ▒ 小地图(minimap)   状态(screen_status)        │
   │           ⎯⎯ 握把(screen_grab) ⎯⎯               │
   └──────────────────────────────────────────────────┘  ← 底边投影
```

- **打开入口**：标题栏菜单 →「查看屏幕」。
- **打开 = 开始采集**，收起 = 停止采集。
- **平移模型**：`translationY` 在 `[-panelHeight, 0]` 间滑动。
- **收起即 GONE**：SurfaceView 独立合成层，`INVISIBLE` 约束不住泄露，必须 `GONE`。
- **首帧前盖板**：`screen_cover` 盖住透明 SurfaceView，首帧后隐藏。

### 9.2 开合手势

| 手势 | 触发区 | 行为 |
|------|--------|------|
| 下拉打开 | 标题区（headerDragHost） | 仅在未打开时接管；下滑 > touchSlop 跟手 |
| 顶部握把收起 | screen_grab（顶部 64dp） | 上滑收起、下滑展开，带 VelocityTracker |
| 底部热区收起 | screen_collapse_zone | 仅在已打开时接管；上滑收起 |

- **松手吸附**：速度 `vy > 800` → 开；`vy < -800` → 合；否则 `progress > 0.4` → 开。
- **动画**：`280ms`，`DecelerateInterpolator(1.4f)`。

### 9.3 键盘联动 — 窗帘式上推

呼出文字输入栏并弹出键盘时，幕布整块向上平移（画面尺寸不变）。

- `keyboardLift` = `panel.bottom − inputTop`，逐帧同 IME 动画曲线。
- 收起时 `lift` 收敛到 `0`。

### 9.4 远程画面操作

- **视口**：请求子区域 = 渲染区物理像素 ÷ 手机 density（retina≈2），SCALE_ASPECT_FIT 上采样。
- **主画面拖动**：两轴共用 `dispScale`。
- **小地图**：窗口尺寸以 `screen-meta` 为权威；可拖动红框；长按弹出位置菜单（§9.5）。
- **缩放菜单**：主画面长按弹出缩放选项（1:1 / 0.9 / 0.8 / 0.75），菜单位置跟随长按触点，弹出时有触感反馈。
- **缩放过渡**：见 [远程桌面缩放闪烁文档](remote-screen-resize-flash.md)。

### 9.5 小地图与缩放菜单交互

- **小地图红框拖动**：在红框内按下拖动移动视口，不触发长按菜单。
- **小地图长按菜单**：在红框外区域长按，弹出位置选择菜单（左上/右上/左侧中/右侧中/左下/右下）。
- **缩放菜单**：主画面长按（无拖动时），菜单出现在触点上方，超出屏幕时向左偏移。

### 9.6 关键参数

| 参数 | 值 |
|------|-----|
| 开合动画时长 | `280ms` |
| settle 速度阈值 | `±800` px/s |
| settle 位置阈值 | `progress > 0.4` |
| 握把高度 | `64dp` |

## 10. 文本输入模式

### 10.1 视觉结构
```
┌──────────────────────────────────┐
│  [mic]  [  输入文字…  ]  [send] │
└──────────────────────────────────┘
   ↑                          ↑
  语音切换                   发送
```

### 10.2 手势

| 手势 | 触发区 | 行为 |
|------|--------|------|
| 上拉呼出 | `drag_handle`（全宽感应层） | 拖 > `pullMax × 0.55` → 展开，语音框上移渐出 |
| 轻点呼出 | `drag_handle`，落在 composer_card 内 | 同上 |
| 下滑收起 | EditText 上 | 拖 > 28dp → 切回语音面板 |
| 上滑关幕布 | EditText 上 | 拖 > 28dp + 幕布打开 → 收起幕布 |

- 方向优先：MOVE 阶段确认方向后才拦截，否则透传 EditText。

### 10.3 键盘联动
- 键盘收起时自动 `hideTextInput()`

## 11. 方向交互模式 — D-pad

桌面端发送 `session-input-lost` 时自动切换到方向模式。

### 11.1 按钮重定义

| 按钮 | 正常功能 | 方向模式功能 | 发送命令 |
|------|---------|-------------|---------|
| 中间 | 🎤 语音 | D-pad 四向点击 | `cursorUp/Down/Left/Right` |
| 左侧 | ⌫ 退格 | Esc | `escape` |
| 右侧 | ⏎ 回车 | Enter | `enter` |

方向模式下的 Enter/Esc 不经过 `isInputAvailable()` 检查，直接通过 DataChannel `action` 消息发送至桌面端。

### 11.2 D-pad 判定
- 触点相对按钮中心的象限：`|dy| > |dx|` → 上下；否则 → 左右
- 每次点击发一个方向 + `KEYBOARD_TAP` 触感

## 13. Android 代码架构

```
ChatActivity
├── Lifecycle            — onCreate / resume / pause / destroy
├── Voice Composer       — bindVoiceButton + cursor drag/swipe
├── Composer Buttons     — backspace/enter + hold progress
├── Text Input Panel     — drag-to-open / swipe / send
├── Screen Curtain       — setup / keyboardLift / collapseZone
├── Input State          — normal ↔ D-pad mode
```
