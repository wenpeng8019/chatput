# Chatput Project Guidelines

## 核心规则

### 1. 被否定或无变化即验证前提
- 用户说"你错了"超过 1 次，或改动 2 次以上用户说"没变化/还是一样的" →
  **立即停手**，不要继续在同一个局部改代码。
- 跳出当前假设和前提，检查更上层的原因（平台对不对、文件对不对、公式对不对、
  缓存是否生效、是否被其他代码覆盖、是否被父层级约束或重载等。
- **用户的肉眼观察 > 你的代码推断。**
  此时先验证构建和安装的程序版本
→ 详见 PITFALL.md 案例一、三


### 2. 批量改动再编译
不要每改一行就编译一次。攒到一批完整的改动再构建。

### 3. 确认平台再改代码
改涉及文件前，确认你在正确的平台（Android vs iOS）和正确的文件上操作。
→ 详见 PITFALL.md 案例一

### 4. 改公式/算法先确认正确性
涉及公式、比例、坐标等数学计算时，先确认公式来源（官方文档/源码），
不要靠"自认显然对"就下结论。
